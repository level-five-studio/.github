# Communication & Planning

We centralise work tracking in **GitHub** and direct communication in **WhatsApp**.

---

## GitHub — Everything Code-Related

GitHub is the single source of truth for all work.

| Feature | How We Use It |
|---|---|
| **Issues** | Bugs, features, tasks, and ideas — labelled and milestoned |
| **Milestones** | Group issues into time-boxed delivery goals |
| **Projects (Roadmap view)** | High-level roadmap visualisation |
| **Assignments** | Every open issue/PR has an owner |
| **Wikis** | Long-form reference documentation per repository |
| **GitHub Pages** | Published docs and static sites |
| **Discussions** | Async technical discussions that don't fit in an issue |
| **Actions** | CI/CD, automation, and org-level housekeeping |

**Rule:** if it's about code, a feature, a bug, or a process — it lives in GitHub.

Learn more: [GitHub Projects](https://docs.github.com/en/issues/planning-and-tracking-with-projects) · [GitHub Issues](https://docs.github.com/en/issues)

---

## WhatsApp Community — Direct Communication

Real-time and async communication happens in our **WhatsApp Community**.  
Each topic has a dedicated group to keep conversations focused:

| Group | Purpose |
|---|---|
| **General** | Ad-hoc team communication |
| **Daily / Weekly Updates** | Standups and weekly summaries |
| **Planning** | Sprint and roadmap discussions |
| **Releases** | Release announcements and deployment status |
| **Incidents** | Live incident coordination |

**Rule:** decisions made in WhatsApp that affect the codebase must be captured as a GitHub issue or comment before they are closed.

---

## Rhythms

| Cadence | Format | Where |
|---|---|---|
| Daily check-in | Short async update | WhatsApp — Daily Updates group |
| Weekly sync | Walkthrough of open issues, blockers, upcoming milestones | WhatsApp or video call |
| Milestone review | Demo + retrospective at end of each milestone | Video call |
